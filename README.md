## üß™ Challenges Faced & Solutions Implemented

### **Challenge 1: Complex API Structuring**  
**Solution:** Adopted a modular architecture, separating concerns using DRF ViewSets and Serializer classes.

### **Challenge 2: Understanding Docker and Environment Isolation**  
**Solution:** Practiced writing Dockerfiles, used Docker Compose for multi-container setups, and configured environment variables securely.

### **Challenge 3: CI/CD Failures**  
**Solution:** Implemented step-by-step GitHub Actions workflows, added proper test suites, and debugged pipeline errors systematically.

### **Challenge 4: Managing Asynchronous Tasks**  
**Solution:** Integrated Celery with RabbitMQ and structured background workers for email notifications and heavy tasks.

## üß≠ Best Practices & Personal Takeaways
- Write clean and maintainable code using PEP 8 standards  
- Use environment variables and avoid hardcoding secrets  
- Push small, frequent commits with meaningful messages  
- Document APIs and system architectures  
- Test early, test often  
- Collaborate effectively across backend and frontend roles  
- Always plan your system design before coding
